How it works

Zero config. Full intelligence.

You add one component. VoiceLayer figures out everything else.

01

SDK reads your navigation

VoiceLayer scans React Navigation state (or the DOM for web) to discover every screen automatically — no action definitions needed.

02

Claude maps speech → screen

Your screen names go to Claude. It maps "customers dikhao", "order 1234 dikhao", or "wapas jaao" to the right destination using its own language understanding.

03

Navigate with full context

VoiceLayer knows where the user is, where they've been, and what they just said. Navigation carries params. Pronouns resolve. Multi-turn just works.

04

Opt in for deeper intelligence

Add useVoiceLayerScreen to any screen for richer context. One hook, 3-5 hint phrases — chips and intent get dramatically smarter on that screen.

Intelligence layers

7 layers. All automatic.

Each layer adds a dimension of understanding. Most are zero-config. One is optional for developers who want fine-grained control.

Automatic
1

Screen auto-discovery

Reads React Navigation state on every command to find all reachable screens. Tab screens appear immediately. Stack screens appear after first visit.

// SDK reads navigation state →
Home, Shops, Staff, Meals
// No config needed
Automatic
2

Current screen context

Claude always knows which screen the user is on. "Add karo" on Shops means create a shop. On Customers, it means add a customer. No ambiguity.

Current screen: ShopsScreen ← user is here
User: "add karo"
→ CreateNewShopScreen
Optional hint
3

Param annotations

Annotate screen names with the params they accept. VoiceLayer extracts values from the transcript and navigates with the right params automatically.

screenNames={['OrderScreen:orderId']}

User: "order 1234 dikhao"
→ navigate(OrderScreen, { orderId: '1234' })
Automatic
4

Context-aware chips

Quick-action chips change based on where the user is. On Shops: sibling tabs. On a screen with registered hints: those hints appear first. Always relevant, never stale.

On ShopsScreen →
[Add Shop] [Search] [Home]
On Dashboard →
[Customers] [Earnings] [Staff]
Automatic
5

Navigation history

VoiceLayer tracks every screen transition. "Wapas jaao", "go back", "pehle wali screen" all work naturally. Claude also sees the breadcrumb trail for richer context.

Path: Dashboard → Shops → Customers
User: "wapas jaao"
→ navigationRef.goBack()
Optional
6

Screen hints & elements

One hook, one array. Call useVoiceLayerScreen in any screen to tell VoiceLayer what it can do and what's visible. Chips become specific. Intent becomes precise.

useVoiceLayerScreen({
  hints: ['add shop', 'search'],
  elements: ['Shop list: 3']
})
Automatic
7

Multi-turn conversation

VoiceLayer keeps a rolling window of the last 3 exchanges. Pronouns resolve across commands. References persist. Users speak naturally, not robotically.

User: "customers dikhao" → navigates
User: "unhe naam se sort karo"
"unhe" = customers (from history)
→ CustomerScreen, sort: 'name'
Integration

Ship voice in minutes

Choose your platform. Copy the code. Done.

1

Drop one script tag before </body>

VoiceLayer scans the DOM, detects navigation and interactive elements, and adds a floating mic button. Nothing else needed.

index.html
<!-- Option A: Direct (key in browser) -->
<script
  src="https://unpkg.com/voicelayer-sdk/dist/voicelayer.iife.js"
  data-anthropic-key="sk-ant-YOUR_KEY"
  data-language="hi"
></script>

<!-- Option B: Server proxy (recommended for production) -->
<script
  src="https://unpkg.com/voicelayer-sdk/dist/voicelayer.iife.js"
  data-proxy-url="https://your-server.com/proxy"
  data-language="hi"
></script>
2

Or install via npm (React, Vue, Next.js…)

Import and call init() once. VoiceLayer auto-scans the DOM on every route change.

terminal
npm install voicelayer-sdk
app.js
import { VoiceLayer } from 'voicelayer-sdk'

VoiceLayer.init({
  proxyUrl: 'https://your-server.com/proxy',
  language: 'hi',   // 'hi' | 'en' | 'auto'
  appId:    'my-app',
})
✦

Run the proxy server — API keys stay off the browser

Deploy our open-source analytics server. Unlocks the founder dashboard, usage analytics, and keeps your Anthropic key server-side.

terminal
npx voicelayer-server   # starts on :3001
# Set ANTHROPIC_API_KEY in .env — done.

1

Install the package

One peer dependency: @react-navigation/native — you almost certainly have this already. No native modules to link.

terminal
npm install voicelayer-rn
2

Export a navigation ref — two lines you may already have

VoiceLayer uses this to read navigation state and navigate programmatically.

src/navigation/navigationRef.js
import { createNavigationContainerRef } from '@react-navigation/native'
export const navigationRef = createNavigationContainerRef()
3

Drop one component. That's the entire integration.

VoiceLayer auto-discovers every screen, builds context-aware chips, tracks navigation history, and handles all routing. Your app code is completely untouched.

AppNavigator.js
import { VoiceLayerButton } from 'voicelayer-rn'
import { navigationRef }   from './navigationRef'

export default function AppNavigator() {
  return (
    <NavigationContainer ref={navigationRef}>
      {/* your existing navigators — completely untouched */}

      <VoiceLayerButton
        navigationRef={navigationRef}
        apiKey="vl-your-key-here"
      />
    </NavigationContainer>
  )
}
✦

Have push-only or parameterised screens?

Stack screens not yet visited won't be in nav state. Annotate their names once — Screen:param syntax. That's all the config VoiceLayer ever asks for.

optional — push screens & deep params
<VoiceLayerButton
  navigationRef={navigationRef}
  apiKey="vl-your-key-here"
  screenNames={[
    'CheckoutScreen',              // push-only, no params
    'OrderDetailScreen:orderId',   // "order 1234 dikhao" → { orderId:'1234' }
    'CustomerScreen:customerId',
  ]}
/>
✦

Optional: richer context on a specific screen

Add useVoiceLayerScreen to any screen for smarter chips and intent. Completely optional — the SDK works without it on every screen.

screens/ShopsScreen.jsx — optional
import { useVoiceLayerScreen } from 'voicelayer-rn'

export default function ShopsScreen() {
  useVoiceLayerScreen({
    hints: ['add shop', 'search shops', 'edit shop'],
  })
  // rest of your screen — unchanged
}
